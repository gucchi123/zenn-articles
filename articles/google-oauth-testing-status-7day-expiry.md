個人でYouTube Data APIを使った自動化ツールを運用していると、**なぜか週1ペースで再ログイン（OAuth同意画面）を求められる**という現象に悩まされていました。リフレッシュトークンを保存して自動更新する実装は入れているのに、なぜか定期的に失効する。原因を探ったところ、Google Cloud側の設定に単純な見落としがありました。

## 症状

`google-auth-oauthlib` でリフレッシュトークンを保存し、期限が近づいたら `refresh_token` で自動更新するコードは実装済みでした。

```python
creds = Credentials.from_authorized_user_file(token_path, SCOPES)
if creds and creds.expired and creds.refresh_token:
    creds.refresh(Request())  # ここで新しい access_token が取れるはず
```

にもかかわらず、**7日前後の周期で** `refresh()` が失敗し、`InstalledAppFlow.run_local_server()` によるブラウザ経由の再認証が必要になっていました。

## 原因: OAuth同意画面の「公開ステータス」

Google Cloud Consoleの「Google Auth Platform」→「対象 (Audience)」を確認したところ、公開ステータスが **「テスト中 (Testing)」** のままになっていました。

これが直接の原因です。Google Cloudでは、OAuth同意画面が「テスト中」ステータスの場合、**発行されたリフレッシュトークンが7日間で自動的に失効する**という制限があります。これは開発中のアプリが誤って本番運用されるのを防ぐための仕様で、ドキュメント上には明記されていますが、「動いているから大丈夫」という思い込みで見落としがちなポイントです。

対象スコープ（YouTube系のスコープは「制限付きスコープ」に分類されます）によっては、Googleの審査（アプリ確認）を通さないと「アプリを公開」できないと思われがちですが、**個人の自分自身のアカウントに対してだけ使うツールであれば、審査無しで「公開」ステータスに切り替えても問題なく動作します**。切り替え後は「Google で確認されていないアプリです」という警告画面が同意時に一度出ますが、それを許可すれば通常通り認可が完了し、以降はリフレッシュトークンが正常に長期間有効になります。

## 対処

1. Google Cloud Console → 該当プロジェクト → 「Google Auth Platform」→「対象」を開く
2. 「公開ステータス」が「テスト中」になっていれば、「アプリを公開」をクリック
3. 個人利用のみであれば、審査プロセスに進む必要はなく、この操作だけで完了

## 教訓

- 「なぜか定期的に再認証が必要になる」系のOAuthの不具合は、まず**OAuth同意画面の公開ステータス**を疑う
- 制限付きスコープ（YouTube, Gmail等)を使っていても、**自分専用のツールなら「アプリを公開」だけで審査無しに恒久化できる**（審査が必要なのは、不特定多数の外部ユーザーに使わせる場合）
- 複数のGoogle Cloudプロジェクトを併用している場合（APIクォータ対策の予備プロジェクトなど）、**プロジェクトごとに公開ステータスを確認する**必要がある。片方だけ本番化して、もう片方を放置していると同じ問題が再発する
